Minggu ini menandakan sebulan menjadi DevOps Engineer di tempat ku magang, di Divisi ku yaitu IT terdiri dari 5 orang pertama ada Mentorku yang berperan sebagai Software Engineer yang memustukan stack dan infra yang ingin dipakai kemudian mentor kedua berperan sebagai Full Stack serta Project Management, kemudian diikuti oleh Backend dan Full Stack Developer (dia juga pegang frontend) kemudian aku menjadi opersional dan infrastuktur. Sayangnya temen ku yang isi Front End diterima kerja di tempat lain dan hanya menemani selama 3 minggu terhitung dari onboarding, jadi timnya hanya tinggal 4 orang dan kedepan aku mungkin akan berperan juga dalam development aplikasi karena PIC serta mentorku bilang kalau bisa bantu pengembangan karena kurang tenaga. AKu pribadi tidak keberatan karena sejauh ini menjadi DevOps tidak banyak tuntutat serta aku banyak menghabiskan waktu ku untuk membaca dokumentasi. 

Tugas pertama yang diberikan adalah menyiapkan infrastruktur untuk host aplikasi yang sedang dikembangkan, infrastruktur yang dipilih adalah Cloud Computing dengan Google Cloud Platform sebagai providernya karena memberikan credit gratis selama 3 bulan. Infra yang dipakai adalah Compute Engine dengan 2 Instance bertenaga 2Vcore, 4GB Ram, serta 30Gb karena masih coba coba sempet berdiskusi untuk mencoba komputer yang lebih bagus namun aplikasi yang dihost sekarang tidak perlu tenaga banyak jadi akan memakai apa yang dipunya.

Tugas kedua ku adalah membuatkan Dockerfile dan Image untuk aplikasi, ku akui sudah lama tidak memakai Docker lalu terakhir kali memakai Docker di mesin Windows, pengalamannya tidak menyenangkan karena begitu sulit untuk menjalankan enginenya sehingga [aku beralih ke Linux]({% post_url 2025-10-13-kenapa-biaya-perbaikan-laptop-ku-mahal-sekali %}) untuk memakai Docker. Kali ini aku mencoba lagi dengan langsung install Docker Engine di dalam WSL2 bukan lewat Docker Dekstop dengan mengikuti [instruksi instalasi pada mesin Ubuntu](https://docs.docker.com/engine/install/ubuntu/) hasilnya Docker daemon bisa dipakai dan digunakan untuk membuat image aplikasi selain Docker mentorku juga ingin mencoba Podman karena ingin tahu perbedaanya apa, yang ku perhatikan adalah Podman itu daemonless jadi tidak berjalan dengan systemd sehingga jika terjadi crash pada systemd maka container tidak akan terpengaruh kemudian rootless jadi tidak perlu memakai perintah `sudo` untuk menjalankan perintah Podman. Berbeda dengan Docker yang perlu `sudo docker run hello-world` untuk menjalankan container sampai menambah User Docker kedalam group lalu terakhir adalah build engine yang berbeda. DOcker menggunakan buildx dan bake untuk membangun image dan cache, Podman sendiri menggunakan buildah untuk membangun image yang kubaca sekilas dari dokumentasi dan [blog](https://podman.io/blogs/2018/10/31/podman-buildah-relationship)

Dengan Docker dan GCP maka bagian dari Deployment dan Operate sudah ada dalam cycle DevOps, sejauh ini dalam bagian infra aku ditugaskna untuk host aplikasi di dalam instance dengan Container. Saat ini sudah ada 2 Container yang ada di dalam instancek kemudian telah juga dibuatkan reverse proxy untuk kedua service tersebut dengan Nginx lalu dibuatkan juga HTTPS koneksi dan SSL Certificate. Menyiapkan networking di dalam GCP adalah bagian yang menarik dari proses membuat infrastukrtur karena mencoba menyambungkan service dengan internet itu harus memerhatikan firewall rule dan port yang perlu dibuka kemudian port yang dibuka aplikasi harus sesuai dengan port yang dibuka pada host machine. 

Lalu selama menyiapkan infrastruktur aku belajar dan mempraktekan SSH Key untuk menyiapkan SSH autentikasi agar bisa terkoneksi dengan instance tanpa perlu membuka Google Console, serta service sepert Gitlab dan ternyata kegunaannya lebih dari itu.
