Selama 2 minggu terakhir di bulan April aku menjalankan misi yaitu mencoba membuat game server di dalam container, aku berpikir melakukan ini setelah magang 6 bulan menjadi DevOps Engineer dan merasa kalau game server di dalam container bisa dilakukan selama depedensi dan cara memulai gamenya bisa diatur di dalam [Dockerfile](https://docs.docker.com/reference/dockerfile/). 

Aku mengikuti guide milik [Harry Potter](https://github.com/fbef0102/Sourcemod-Server) dalam instalasi game server, aku pertama kali mencobanya di laptop lama untuk menjalankan game server. Di guide yang kutemukan di internet *user* bisa mendownload secara anonim lewat `steamcmd` namun sekarang tidak bisa dan harus login terlebih dahulu, aku melihat [forum di steam](https://steamcommunity.com/app/550/discussions/0/599642275953250985/) untuk alternatif lainnya adalah menggunakan DepotDownloader dan ketika aku coba pakai buat download gamenya ternyata berhasil dan game servernya sudah bisa dijalankan kemudian aku bisa terkoneksi ke dalam server ku walaupun local.

Sampai tahap ini aku masih mencoba secara langsung di laptop lama tahap berikutnya adalah mencoba packaging L4D2 Servernya ke dalam container, Selama aku membuat aku tidak tahu kalau [`steamcmd` sudah ada Docker Imagenya](https://hub.docker.com/r/cm2network/steamcmd) sehingga selama aku membuat imagenya masih belum mengetahui apa saja yang diperlukan oleh gamenya sehingga ada beberapa error seperti game tidak mau jalan, game tidak terdeteksi, ataupun steamclientnya tidak terkoneksi. Karena itu aku mengikuti instalasi `steamcmd`dari [repository CM2WAIKI](https://github.com/CM2Walki/steamcmd/blob/master/trixie/Dockerfile) mulai dari konfigurasi steam user sampai home usernya.

selama proses unduh dan instalasi game serta steamcmdnya masih sama namun ketika dijalankan terjadi error yaitu steamclientnya tidak bisa dijalakan dari execstack. Dari [forum steam ini](https://steamcommunity.com/app/224260/discussions/2/565867433751020858/) dijelaskan kalau Valve update runtime dari library mereka kemudian sistem operasi yang terbaru saat ini mencegah kode eksekusi di dalam stack karena dikira adalah *malicious behavior* demi keamanan sistem. Jalan keluarnya adalah menggunakan `pathelf` untuk clear execstack dari binary yang dipakai.

Setelah itu selesai game berhasil dijalankan dan aku berhasil terhubung dengan servernya ku yang ada di dalam container, aku akan jelaskan ini di [dokumentasi nya tersendiri](/homelab/gameserver) bagaimana pendekatan dan cara kerjanya di sini aku merasa bahwa 80% pekerjaan untuk menjalankan game server di dalam container sudah selesai. 20% nya adalah menambahkan plugin, membuat server configuration, membuat Dockerfilenya user friendly dan berbagai macam tahapan yang berada di *downstream* nantinya.

Mari bicara mod dalam server, pada sourcegame modding dan scripting yang populer adalah [Sourcemod](https://www.sourcemod.net/about.php) dan [Metamod](https://www.metamodsource.net/about). Valve merilis gamenya dengan dua versi yaitu Linux dan Windows, Sourcemod dan Metamod memiliki dua versi untuk sistem operasi Linux dan Windows. Pada tahap instalasi mengalami kendala yaitu salah memasang versi dari SM dan MM, karena server yang kujalankan adalah Linux maka aku perlukan binary untuk Linux sedangkan yang aku pakai adalah versi Windows, aku tidak sadar ini karena aku hanya copy paste dari repository dan tidak menyadari pesan di *readme* nya yang tertulis kalau plugins ini untuk Windows. Aku baru sadar ketika melihat bahwa extensi dari binarynya itu `.dll` dan untuk Linux adalah `.so` itulah kenapa ada dua tipe download di website mereka.

Setelah instalasi SM dan MM hal yang sama sulitnya adalah mencari plugins yang support platfrom yang dipakai untuk server, karena tidak semua plugins yang dicompile untuk Windows bisa dipakai untuk Linux seperti plugins [l4dtoolz](https://github.com/lakwsh/l4dtoolz) plugins ini berfungsi untuk mengatur jumlah player serta tick rate di dalam server. Di dalam releasenya ada dua versi yaitu untuk Windows dan Linux, namun ada beberapa plugins yang tidak release versi Linux nya sehingga harus build from source untuk bisa mendapatkan pluginsnya.

Souremod memiliki extensi khususnya yaitu `.smx` untuk plugins yang dipakai, dari observasi ku tentang direktori struktur sourcemod ada scripting dan include, scripting ini adalah bahasa pemrograman yang dipakai untuk membuat plugins kemudian include sebagai tempat untuk data struktur yang akan dipakai oleh scripting. Setelah membuat plugins yang inginkan menggunakan sourcespawn berikutnya adalah compile, sourcemod menyediakan compile dengan Windows maupun Linux yang aku belum tahu apakah plugins dari SM terpengaruhi jika di compile oleh Windows maupun Linux. 

Dan masih banyak lagi yang aku belum ketaui yaitu extension untuk sourcemod fungsinya untuk apa lalu metamod memiliki formatnya sendiri dengan `.vdf` dan aku belum tahu ini fungsinya apa, yang jelas mendapatkan plugins ataupun menulis sendiri plugins itu lebih sulit dibandingkan start game server. Menurutku orang awam yang ingin hosting servernya sendiri tanpa plugins apapun bisa selesai dalam 20 menit sedangkan plugins bisa menghabiskan seharian karena faktor faktor seperti extension yang diperlukan, apakah untuk Linux atau Windows, lalu data data lain yang diperlukan agar plugins itu bisa bekerja memperlukan resource lebih banyak dibandingkan memulai server.
